Quickstart

The bare minimum to get csv-lsp running: install the binary, tell your editor about it, and take the sixty-second tour.

What you get

  • Live diagnostics — rows with missing/extra cells, unclosed quotes, stray quotes, text after a closing quote. Exact spans, updated on every keystroke.
  • Code actions (Spacea in Helix) — pad short rows, align/compact columns, reinterpret or convert the dialect, quote cells/columns, add/delete columns.
  • Formatting (:format) — align columns so delimiters line up; idempotent, so format-on-save is safe.
  • Column selection (Spaceh) — one cursor per cell of the column under your cursor.
  • Unicode-aware alignment, BOM/CRLF/final-newline preservation, multi-line quoted cells, and four dialects: CSV, TSV, SSV (semicolon), PSV (pipe).

The Features page walks through each of these with examples and GIFs.

Install

git clone https://github.com/tammoippen/csv-lsp
cd csv-lsp
cargo install --path .

Requires a stable Rust toolchain (edition 2024; the manifest declares rust-version = "1.85"). The binary lands in ~/.cargo/bin/csv-lsp — make sure that directory is on your PATH, then check:

csv-lsp --help 2>/dev/null; which csv-lsp   # any output path means it is installed
Rust concept · cargo install

cargo install --path . builds the crate’s binary target in release mode and copies it into ~/.cargo/bin. The project commits its Cargo.lock, and cargo install respects a committed lockfile for path installs — so you build exactly the dependency versions the repository was tested with. There is nothing else to deploy: an LSP server is just an executable that reads stdin and writes stdout.

Helix setup

Add this to ~/.config/helix/languages.toml. It registers the server and the four file types; nothing else is required for diagnostics, code actions and :format:

[language-server.csv-lsp]
command = "csv-lsp"

[[language]]
name = "csv"
scope = "text.csv"
file-types = ["csv"]
language-servers = ["csv-lsp"]
auto-format = false            # true = align on every save

[[language]]
name = "tsv"
scope = "text.tsv"
file-types = ["tsv", "tab"]
language-servers = ["csv-lsp"]
grammar = "csv"                # tsv/ssv/psv reuse the csv grammar
auto-format = false

[[language]]
name = "ssv"
scope = "text.ssv"
file-types = ["ssv"]
language-servers = ["csv-lsp"]
grammar = "csv"
auto-format = false

[[language]]
name = "psv"
scope = "text.psv"
file-types = ["psv"]
language-servers = ["csv-lsp"]
grammar = "csv"
auto-format = false

Verify with hx --health csv — the language server line should show csv-lsp as found.

Optional · rainbow columns

Column colors come from a tree-sitter grammar, not from the LSP server (Helix highlights exclusively through tree-sitter). This repository ships an improved grammar in tree-sitter-rainbow-csv/ that fixes empty-cell and TSV handling. Setup takes two extra steps — see Features → Rainbow columns.

The sixty-second tour

Create a deliberately broken file and open it:

cat > demo.csv <<'EOF'
id,name,qty,price
1,bolt,100,0.10
2,nut,250
3,"washer,steel",75,0.05
4,screw
EOF
hx demo.csv
  1. Diagnostics appear immediately. Rows 3 and 5 are short — each gets an error at its end: row-missing-cells “row has 3 cells, expected 4”. Press Spaced for the diagnostic picker, or hop with ]d / [d.
  2. Fix one row. Cursor on the 2,nut,250 row, Spacea, pick Pad row with 1 empty cell. The row becomes 2,nut,250, — a trailing comma is an empty fourth cell. Or pick Pad all short rows (2) to repair the whole file at once.
  3. Align. :format pads cells with spaces so the delimiters line up: 
    id,name          ,qty,price
1 ,bolt          ,100,0.10
2 ,nut           ,250,
3 ,"washer,steel",75 ,0.05
4 ,screw         ,   ,
    The comma inside "washer,steel" is quoted content, not a delimiter — the parser knows the difference.
  4. Compact. Spacea → Compact columns strips the padding again, byte-for-byte back to the compact form. Padding is layout, never data.
  5. Select a column. Cursor anywhere in the name column, Spaceh — one selection per cell. Type c and every cell of the column changes at once.

Dialects and detection

DialectDelimiterExtensionsTypical origin
csv,.csvthe default everywhere
tsvtab.tsv, .tabdatabase dumps, spreadsheets
ssv;.ssvEuropean-locale exports (, is the decimal separator)
psv|.psvdatabase and log exports

The dialect is resolved once, when a file is opened, with this precedence:

  1. the LSP languageId the editor sends (Helix: the name of the matching [[language]] entry),
  2. the file extension,
  3. content sniffing — delimiters counted outside quotes in the first non-blank line, ties favor comma,
  4. CSV as the final default.

For files whose extension lies (a .csv that is actually semicolon-separated), use Reinterpret as SSV (parse differently, zero text changes, session-scoped) or Convert to CSV (rewrite the text, quoting adapts). Details in Features → Reinterpret and → Convert.

Diagnostics reference

CodeMeaningSeverityFix
row-missing-cellsfewer cells than the headererrorquickfix: pad with empty cells
row-extra-cellsmore cells than the headererrormanual (deleting data is not auto-fixed)
unclosed-quotequoted cell never closederrormanual
stray-quotebare " in an unquoted cellwarningquote the cell or double the quote
text-after-quotetext after a closing quoteerrormanual

Two conventions worth knowing, because they explain most “why is there no error here?” moments: the header is the first non-blank row and its cell count is the column contract; and rows the parser could not fully understand (quoting errors) are excluded from column checks — one unclosed quote must not cascade into a wall of ragged-row errors.

Logging

stdout is the protocol channel, so logs go to stderr, gated by an environment variable:

CSV_LSP_LOG=1 hx -v data.csv    # then check ~/.cache/helix/helix.log

Other editors

csv-lsp is a standard LSP server over stdio — any client works. You need to tell your editor two things: run the command csv-lsp, and attach it to the file types csv/tsv/ssv/psv (sending one of those strings as the languageId picks the dialect directly; otherwise the file extension decides). For Neovim that is an vim.lsp.start call in a FileType autocommand; for VS Code you need any generic-LSP-client extension. The server negotiates its position encoding automatically (UTF-8 preferred, UTF-16 fallback) and requires no configuration.

Troubleshooting

  • No diagnostics at all — run hx --health csv; if the server binary is not found, check that ~/.cargo/bin is on the PATH Helix sees. Then CSV_LSP_LOG=1 hx -v file.csv and look for initialized with … in the Helix log.
  • Everything is one column / diagnostics look wrong — the extension probably lies about the delimiter. Spacea → Reinterpret as … fixes the session; renaming or converting the file fixes it durably.
  • Rainbow colors are off or missing — colors come from the tree-sitter grammar, not the server; see Features → Rainbow columns for the grammar override and its one cosmetic caveat.
  • Worried about format-on-save loops — don’t be: formatting an already-aligned file returns no edits by construction, so auto-format = true is safe.
